$ git init [ім'я директорії] - ініціює пустий репозиторій в поточній директорії. Так само можна вказати будь-який шлях $ git status - провірка репозиторію $ git add [ім'я файла] / $ git add . - запис змін в репозиторій даного файла / всіх файлів $ git commit [ім'я файла] / $ git commit -m "Текс коміту" - збереження поточного весрії файла $ git log - показує всі зміни які відубвались з даним репозиторієм $ git checkout - премикання між наявними вітками репозиторія $ git branch - перевірка назви вітки репозиторія і провірка доступних віток репозиторія / створення нової вітки репозиторію $ git push - виввести репозиторій в github $ git pull - з онлайн репозиторія перенести всі файли, що нема в офлайн репозиторії $ git merge [навза вітки] - об'єднує дві вітки репозиторія в одну і показує чи є конфлікти між файлами (заносить всі файли із [назви візитки] у вітку де ми знайходимось) $ git clone [посилання / директорія]- робить локальну копію для роботи із репозиторієм
